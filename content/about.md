I am Ben Gauger. I work in DevOps and infrastructure, and I use my homelab as a place to practice the habits that make systems less fragile.

This blog is where I write down the parts worth remembering: Proxmox mistakes, GitOps patterns, self-hosted services, automation, backups, and the occasional reminder that a green dashboard is not the same thing as a working system.

## What I write about

- Homelab infrastructure
- Proxmox and virtualization
- OpenTofu, Terraform, and infrastructure as code
- Ansible and configuration management
- Docker, LXC, and self-hosted services
- Operational habits that transfer to real production work

The articles are not meant to be perfect universal guides. They are notes from real systems with real constraints. If something worked here, I explain why. If something broke, that is usually the more useful part.

## How this site is built

This site is self-hosted and managed the same way I prefer to manage infrastructure:

- written in Obsidian
- built with Hugo
- deployed through GitHub webhooks
- served through Caddy and Cloudflare Tunnel
- managed with OpenTofu and Ansible where practical

The site is also part of the experiment. Publishing the notes forces me to make the work legible instead of leaving it trapped in terminal history and half-remembered fixes.
