This site is mostly notes from running a homelab like a small production environment.

Not because the basement rack needs enterprise ceremony. It does not. But because the same habits keep showing up: write changes down, automate the boring parts, test restores, avoid mystery servers, and do not trust a green dashboard more than it deserves.

## The operating idea

A homelab is a useful place to practice real operational habits with smaller blast radius.

Things break. That is part of the point. The value comes from turning the breakage into a better default, a playbook, a runbook, or at least a note that future me can find before making the same mistake again.
